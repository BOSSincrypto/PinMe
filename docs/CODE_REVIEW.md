# Результаты код-ревью

Проверены Android-приложение, web-прототип, хранение данных, экспорт/импорт и GitHub Actions.

## Исправлено

- Удалён plaintext-экспорт и импорт web-прототипа, включая захардкоженный пароль.
- Для паролей контактов добавлены PBKDF2-SHA256, соль, число итераций и миграция старого формата.
- Android-база переведена на случайный ключ SQLCipher, ключ обёрнут Android Keystore.
- Добавлена безопасная миграция базы со старого статического ключа и резервная копия перед rekey.
- Импорт резервной копии стал строгим: проверяются версия, размеры, идентификаторы, связи и лимит файла.
- Удаление контакта выполняется транзакционно с очисткой зависимых данных и отменой уведомлений.
- Заблокированные контакты исключаются из агрегатов, поиска и экранов дат, напоминаний и разговоров.
- Включён `FLAG_SECURE`, приложение блокируется при уходе в фон, содержимое уведомлений обобщено.
- Добавлены атомарная запись web-хранилища, проверка схемы и обработка quota/storage ошибок.
- Добавлены CI, подписанные Android Release, SHA-256 checksum и Dependabot.

## Проверки

- `npm run lint`
- `npm run typecheck`
- `npm run build`
- `npm audit --audit-level=high`
- `./gradlew lintDebug testDebugUnitTest assembleDebug`
- `actionlint .github/workflows/*.yml`
- Локальная release-сборка с временным keystore, `apksigner verify` и `zipalign`.

## Остаточные ограничения

- Web-прототип по-прежнему хранит данные в `localStorage`; для конфиденциальных данных используйте Android-приложение.
- Уведомления Android создаются из текущего UI-потока; отдельный WorkManager-планировщик для фоновых напоминаний ещё не добавлен.
- Для обновления production APK нужно навсегда сохранить production keystore и его пароли.
- Первая production-установка не обновляет debug APK из-за различий подписей.
- В репозитории пока нет отдельного набора unit-тестов для миграций SQLCipher и криптографических сценариев; CI проверяет сборку и существующие тесты.
