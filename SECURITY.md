# Политика безопасности

Не публикуйте пароли, keystore, резервные копии, содержимое контактов или инструкции по эксплуатации уязвимости в обычном Issue.

Для конфиденциального отчёта используйте раздел **Security → Report a vulnerability** в GitHub-репозитории. Укажите затронутую версию, условия воспроизведения, ожидаемое поведение и минимальный безопасный пример без реальных пользовательских данных.

Поддерживается актуальная версия из последнего GitHub Release. Текущие автоматические APK являются debug-сборками для тестирования; для production-дистрибуции необходим отдельный стабильный keystore.

Открытый JSON-экспорт не является защищённым файлом: резервный пароль только разрешает операцию внутри приложения. Не отправляйте такой файл через небезопасные каналы и удаляйте его после использования.

Никогда не прикладывайте к отчёту:

- production keystore и пароли подписи
- пароль приложения, контакта или резервной копии
- ключ SQLCipher или файлы рабочей базы
- незашифрованный экспорт реальных контактов
